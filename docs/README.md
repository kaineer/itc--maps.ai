# Основная цель

 * Разработка frontend-части приложения для отображения карт.
 * Здания на карте могут отображаться исходя из данных, полученных из openstreet-карт, либо, если в базу загружена модель, соответствующая модель должна отображаться вместо исходного здания.
 * Для отображения зданий на экране используется библиотека @react-three/fiber
 
## Основные режимы работы

* Показ карты, свободное передвижение "по городу"
* Добавление зданий на карту города
  * Здание может быть добавлено с указанием адреса здания, которое оно должно заменить
  * На самом деле, без указания адреса здание не сможет заменить "коробку"
  * При этом должна быть обеспечена возможность "подкорректировать" позицию здания, одновременно показывая здание как "коробку" и как модель
* Режим модерации добавленных зданий
  * Суть в том, что просто добавленная модель еще не может быть добавлена на карту или в экскурсию
  * Сперва ее должны "утвердить", чтобы на карту не попадали плохо сделанные модели
* Редактирование "экскурсий" по городу
  * Экскурсия -- набор точек по которым пользователь должен пройти, чтобы экскурсию закончить
  * Каждая точка имеет координаты и, возможно, список ограничений на возможность движения пользователя (нельзя поворачивать голову, нельзя поднимать голову и т.п.)
  * Также не исключено, что для каждой из точек экскурсии будет назначено расстояние до зданий, которые еще будут грузиться (чтобы не грузить весь город)
  * К точке может быть привязано сообщение c html-разметкой, которое можно будет посмотреть для данной точки
